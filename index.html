Portfolio

  • Penn Global

    I led the development of a set of new sites for the University of Pennsylvania. They required a system that allowed for multiple sites to co-exist in the same Drupal installation without being in a Drupal multisite configuration. To accomplish this, I implemented a taxonomy-based sub-site system. Individual pages could be tagged with the appropriate taxonomy to include them in the desired sub-site.

    Additionally, the new sites had considerable design overlap, so I implemented an inter-theme code sharing framework that allowed sub-sites to branch from a main theme. This system went on to be used in the future to quickly build new University of Pennsylvania websites with similar design overlap.

    Some of the sites included in the framework:

    • Penn Global
    • International Student & Scholar Services
    • Penn Abroad
    • Global Support Services
    • Finance & Administration
    • Student Registration & FInancial Services

  • Eastern Standard Web Frameworks

    While working at Eastern Standard I developed both Wordpress and Drupal frameworks to streamline development efforts across the team.

    Some of the code specific benefits of the frameworks are:

    • Single-folder components
    • Common template base components
    • Pre-configured build tooling
    • Commonly used utility functions
    • Compatibility polyfills
    • Custom plugins/modules
    • Standardized code architecture
    • Defined git workflow

    The frameworks are more than code, they are integrated into Eastern Standard's workflow process. I led meetings where we brought together the leads from design, user experience, and development to discuss changes and improvements to the frameworks. I built an Airtable to organize our project requirements into one central location. I also wrote documentation for developers and non-developers explaining functionality and how they could use the frameworks in their roles.

    Over the years I witnessed other developers attempt to create frameworks that would increase efficiency through a shared system, but all failed to achieve adoption because they were not easy to implement or strayed too far from existing processes. Through these experiences I learned how important ease of adoption is. When I had the chance to implement a system of my own, I focused on the user experience for my colleagues. This was key to the success of my frameworks.

    These frameworks have been in use for years and continue to be the foundation of most projects at Eastern Standard continually gaining efficiency for the team.
